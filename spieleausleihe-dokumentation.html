Systemkonzept

Das System besteht aus vier aufeinander aufbauenden Ebenen. Wer das Prinzip einmal verstanden hat, findet sich in allen Tabs zurecht:

EbeneWas ist das?Bedeutet Besitz?
SpieledatenbankReferenz aller EANs, die wir jemals gescannt haben — Titel, Bild, Spieleranzahl, Alter, Dauer, Bewertung.Nein — reine Nachschlage-Referenz.
SammlungEine benannte Gruppe von Spielen, die jemand tatsächlich besitzt (z.B. "Spieleversum", "Andreas privat", "Genduron").Ja — eine Sammlung sagt aus, wer ein Exemplar besitzt.
EventEine konkrete Veranstaltung, die aus einer oder mehreren Sammlungen Spiele auswählt und in eigene Regale einräumt.Legt fest, was bei dieser Veranstaltung verfügbar ist.
AusleiheDer operative Betrieb: Scannen, Ausleihen, Zurückgeben — immer bezogen auf das gerade aktive Event.Zeigt den Live-Status während der Veranstaltung.

Merksatz: Ein Spiel muss zuerst in der Datenbank stehen, dann einer Sammlung gehören, dann einem Event zugeordnet sein, bevor es überhaupt ausgeliehen werden kann.

Zugriff & Anmeldung

  • Das Verwaltungstool (spieleausleihe-verwaltung.html) erfordert für alle Funktionen eine Anmeldung mit E-Mail und Passwort.
  • Neue Konten können sich über "Noch kein Konto? Registrieren" selbst anlegen — dafür ist ein Zugangscode nötig, den die Vereinsleitung ausgibt.
  • Die öffentliche Übersichtsseite (index.html) benötigt keine Anmeldung — sie zeigt nur, keine Bearbeitung möglich.

Typischer Ablauf vor einem Event

  1. Neue Spiele scannen → Stufe 1 in der Spieledatenbank.
  2. Titel automatisch anreichern lassen (GTNM-Referenz oder Titel-Import) → Stufe 2.
  3. Spiele der passenden Sammlung zuordnen (z.B. "Spieleversum").
  4. Event anlegen, Quell-Sammlungen wählen.
  5. Regale für das Event anlegen (Regal A, B, C, …).
  6. Spiele aus den Sammlungen ins Event holen und Regalen zuweisen.
  7. Event als aktiv setzen.
  8. Am Ausleihtresen: Ausleihe/Rückgabe nutzen.
  9. Einbett-Code auf der Vereins-Website einfügen, damit Besucher live sehen, was verfügbar ist.
  10. Nach dem Event: Statistik auswerten für die nächste Planung.

Spieledatenbank — Stufe 1: EAN erfassen

Tab Spieledatenbank. Hier werden neue Barcodes roh erfasst, bevor sie einen Titel haben.

  • EANs nacheinander scannen (Scanner tippt automatisch in das Textfeld, eine Zeile pro Code) oder als Liste einfügen.
  • Auf "Erfassen" klicken.
  • Das Tool gleicht jede neue EAN automatisch gegen die hinterlegte GTNM-Referenzliste ab (eine Titel-Datenbank mit ca. 18.500 Einträgen). Bei Treffer wird der Titel sofort übernommen und mit Quelle: GTNM-Referenz markiert.
  • Werden Titel gefunden, fragt das Tool nach, ob diese sofort automatisch bei BoardGameGeek (BGG) angereichert werden sollen (Bild, Spieleranzahl, Alter, Dauer, Bewertung). Das läuft mit Fortschrittsbalken, ca. 5 Sekunden pro Spiel (BGG-Rate-Limit).

Ein Treffer in der GTNM-Referenz ist kein Besitznachweis — er liefert nur einen wahrscheinlichen Titel. Bei Karteileichen oder falsch zugeordneten EANs (z.B. Erweiterung statt Grundspiel) im Zweifel manuell nachschärfen (siehe Datenbank bearbeiten).

Spieledatenbank — Stufe 2: Titel-Liste importieren

Für EANs, die weder Scanner-Team noch GTNM-Referenz kennen (z.B. seltene/regionale Ausgaben), kann eine von Claude recherchierte Titel-Liste importiert werden.

  • Format: [{"ean":"4001504xxxxxx","title":"Catan"}] — eine JSON-Liste mit EAN und vermutetem Titel.
  • In das Textfeld einfügen, auf "Import starten" klicken.
  • Für jeden Eintrag wird automatisch bei BGG gesucht (5 Sek. Abstand), das Ergebnis wird übernommen und mit Quelle: Recherche-Import markiert.
  • Der Fortschrittsbalken und das Protokoll darunter zeigen, welche Einträge erfolgreich waren (✅) und welche nicht (⚠️).

Ergebnisse aus Stufe 2 danach immer in der Tabelle stichprobenartig kontrollieren — automatischer BGG-Abgleich wählt bei Mehrdeutigkeit den ersten Suchtreffer, das kann gelegentlich die falsche Edition sein.

Datenbank durchsuchen & bearbeiten

  • Suchfeld filtert nach Name oder EAN, Status-Filter zeigt nur Offene / Fertige / Fehlerhafte Einträge.
  • "(+) Spiel manuell anlegen" öffnet die Eingabemaske für Einzelfälle, inkl. eigener BGG-Suche mit Trefferauswahl.
  • "Bearbeiten" in einer Zeile öffnet dasselbe Formular zum Korrigieren oder Löschen.
  • Status-Badges: Offen (Stufe 1, noch kein Titel/BGG-Daten), Fertig (BGG-Daten vorhanden), Kein Treffer (Titel bekannt, aber BGG fand nichts), Fehler (technisches Problem beim Abgleich).

Sammlungen verwalten

Tab Sammlungen. Eine Sammlung bündelt die Spiele, die eine bestimmte Person oder Gruppe besitzt.

  • Neue Sammlung anlegen: Name (Pflicht) und Beschreibung (optional).
  • Sammlung öffnen → Spiele aus der Datenbank per Name/EAN-Suche hinzufügen. Nur Spiele, die bereits in der Datenbank stehen, können hinzugefügt werden.
  • Dasselbe Spiel (gleiche EAN) kann in mehreren Sammlungen stehen, wenn mehrere Exemplare existieren.
  • Spiele lassen sich aus der Sammlung entfernen, ohne dass sie aus der Datenbank verschwinden.
  • Sammlung löschen entfernt nur die Zuordnung, nicht die Spiele selbst.

Event anlegen

Tab Events. Name, Datum, und Häkchen bei den Sammlungen setzen, aus denen bei diesem Event Spiele ausgewählt werden dürfen.

Ohne mindestens eine ausgewählte Sammlung lassen sich später keine Spiele zum Event hinzufügen — die Auswahl-Liste bliebe leer.

Regale je Event

Jedes Event hat seine eigenen Regale (Regal A, B, C, …) — Regale werden nicht zwischen Events geteilt, da sich der Aufbau von Veranstaltung zu Veranstaltung unterscheiden kann.

  • Event öffnen → Regal-Name eingeben → "Regal anlegen".
  • Ein Regal mit zugeordneten Spielen kann nicht gelöscht werden — zuerst die Spiele umlagern oder entfernen.

Spiele zum Event hinzufügen

  • Im Event-Fenster nach Name oder EAN suchen — es werden nur Spiele aus den für dieses Event gewählten Sammlungen vorgeschlagen.
  • Klick auf einen Treffer fügt das Spiel dem Event hinzu, automatisch im zuerst angelegten Regal (danach über die Dropdown-Auswahl in der Tabelle jederzeit änderbar).
  • Ein Spiel entfernen ist nur möglich, solange es nicht gerade ausgeliehen ist.

Einbett-Code für die Website

Jedes Event hat einen eigenen Einbettungs-Code (<iframe>), sichtbar im geöffneten Event-Fenster. Auf "Code kopieren" klicken und in die Vereins-Website einfügen — zeigt live den Ausleihstatus dieses konkreten Events, ganz ohne Login.

Aktives Event

Oben im Tool wählt eine Dropdown-Liste das aktive Event. Alle Aktionen im Tab "Ausleihe/Rückgabe" beziehen sich ausschließlich auf dieses Event. Die Auswahl wird lokal im Browser gespeichert — jeder Rechner/jedes Gerät am Ausleihtresen muss sein eigenes aktives Event setzen.

Ohne gewähltes aktives Event ist keine Ausleihe möglich — das Scannen zeigt dann einen entsprechenden Hinweis.

Ausleihe & Rückgabe

  • Scannen: EAN in das obere Feld scannen (oder eintippen + Enter). Ist das Spiel dem aktiven Event zugeordnet, wird der Status sofort automatisch umgeschaltet — kein Bestätigungsschritt nötig.
  • Manuelle Suche: Falls der Scanner ausfällt, Spielname im zweiten Feld eintippen, Treffer anklicken.
  • Unbekannte EAN: Ist das Spiel gar nicht in der Datenbank, bietet das Tool an, es direkt anzulegen. Ist es in der Datenbank, aber diesem Event noch nicht zugeordnet, bietet es an, es dem Event hinzuzufügen.
  • Die Tabelle "Aktuell ausgeliehen" zeigt alle offenen Ausleihen mit Regal und Zeitpunkt, inkl. direktem Rückgabe-Button.

Statistik

Tab Statistik. Event auswählen, um Kennzahlen zu sehen: Gesamtzahl Ausleihen, Anzahl verschiedener ausgeliehener Spiele, aktuell noch nicht zurückgegebene Spiele, sowie eine Rangliste der am häufigsten ausgeliehenen Spiele — hilfreich für die Planung, welche Spiele beim nächsten Event unbedingt wieder mitgenommen werden sollten.

Öffentliche Übersicht (index.html)

  • Aufruf mit ?event=EVENT_ID in der URL zeigt direkt dieses Event (so ist auch der Einbett-Code aufgebaut).
  • Ohne Event-Parameter zeigt die Seite ein Dropdown zur Auswahl unter allen vorhandenen Events.
  • Suchfeld, Regal-Filter, Spielerzahl-Filter und Status-Filter helfen Besuchern, ein passendes verfügbares Spiel zu finden.
  • Kein Login nötig, keine Bearbeitung möglich — reine Live-Ansicht.

Glossar

BegriffBedeutung
EANDer Barcode auf der Spielschachtel, eindeutige Kennung eines Spiels in der Datenbank.
GTNM-ReferenzStatische Titel-Nachschlageliste (EAN → Titel) aus einer Händler-Tabelle, nur zur Titel-Erkennung, kein Besitznachweis.
SammlungWer besitzt welches Spiel-Exemplar (z.B. Spieleversum, Genduron, privat).
EventEine konkrete Veranstaltung mit eigenen Regalen und einer Auswahl an Spielen aus bestimmten Sammlungen.
Aktives EventDas Event, auf das sich die Ausleihe/Rückgabe-Funktion gerade bezieht (pro Gerät einstellbar).
Enrich-StatusZeigt, ob ein Datenbankeintrag vollständige BGG-Daten hat (Offen / Fertig / Kein Treffer / Fehler).

Häufige Probleme

ProblemLösung
Scan zeigt "Kein aktives Event gewählt"Oben im Tool ein aktives Event auswählen.
Scan zeigt "Unbekannte EAN"Spiel ist noch nie in der Datenbank erfasst worden → über den angebotenen Button direkt anlegen.
Scan zeigt "Ist in der Datenbank, aber noch nicht diesem Event zugeordnet"Spiel existiert, gehört aber (noch) nicht zum aktiven Event → über den Button direkt hinzufügen.
Regal lässt sich nicht löschenEnthält noch zugeordnete Spiele — zuerst umlagern oder aus dem Event entfernen.
Spiel taucht bei "Spiele zum Event hinzufügen" nicht aufEs gehört zu keiner der für dieses Event ausgewählten Sammlungen, oder es ist noch keiner Sammlung zugeordnet.
BGG-Suche liefert nichtsTitel evtl. zu ungenau/falsch geschrieben — im Bearbeiten-Formular Titel anpassen und BGG-Suche erneut starten.
Registrierung schlägt fehlZugangscode falsch oder E-Mail bereits vergeben — Zugangscode bei der Vereinsleitung erfragen.
